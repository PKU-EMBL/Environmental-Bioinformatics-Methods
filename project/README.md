# 课程汇报工作区 | Presentation Workspace

本目录提供课程汇报的组织建议与规划模板。课程考核只有课程汇报，不另设项目设计书或阶段性项目分数；模板用于帮助团队尽早收敛问题和证据链。

## 推荐目录

```text
presentation-team-name/
├── README.md          # 问题、数据、运行入口与分工
├── data-notes/        # 数据来源、许可、元数据与访问日期
├── env/               # 环境或容器定义
├── workflow/          # 命令、脚本、notebook 或工作流
├── figures/           # 图表、图注与绘图输入
├── references/        # 文献与数据库版本记录
└── slides/            # 最终幻灯片与附录
```

## 最低质量基线

- **问题清楚：** 研究对象、尺度和范围适合课堂汇报。
- **数据可核查：** 来源、accession、许可、质量状态与处理过程明确。
- **方法有理由：** 工具、版本、参数和质量阈值均能对应研究问题。
- **图表有任务：** 每张图服务于一个判断，不堆叠无关输出。
- **解释有边界：** 区分观察、统计、功能推断与生态机制。
- **协作透明：** 小组成员分工、外部资源与 AI 辅助范围如实披露。

建议使用 [课程汇报规划模板](proposal-template.md) 在制作幻灯片前完成一次内部审阅。
